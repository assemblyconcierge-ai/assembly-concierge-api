CHANGES FROM v5 TO v6
======================

1. STRIPE WEBHOOK SIGNATURE VERIFICATION  [COMPLETE]
   File: src/lib/webhookSecurity.ts (rewritten)

   Problem (v5): The webhook route checked the custom X-Webhook-Signature
   header only, using a flat HMAC-SHA256 over the raw body.  Stripe sends
   a stripe-signature header with the format:
       t=<unix_timestamp>,v1=<hex_signature>
   where the signed payload is "<timestamp>.<raw_body>".

   Fix (v6):
   - parseStripeSignatureHeader() parses the t= and v1= fields.
   - verifyStripeSignature() reconstructs the signed payload, enforces a
     300-second replay-window, and does a timing-safe compare.
   - verifyInboundWebhook() is the unified entry point; tries
     stripe-signature first (when STRIPE_WEBHOOK_SECRET is set), then
     falls back to X-Webhook-Signature (Make.com forwarding path).
   - Legacy helpers (signPayload, verifyWebhookSignature) preserved for
     backward compatibility with existing tests.

   New env var: STRIPE_WEBHOOK_SECRET (whsec_... from Stripe Dashboard)
   Existing env var: WEBHOOK_SECRET (used for Make.com forwarding path)

   Route change in src/app.ts:
   - Reads both WEBHOOK_SECRET and STRIPE_WEBHOOK_SECRET at startup.
   - Calls verifyInboundWebhook() instead of verifySignature() directly.
   - Returns { error: 'INVALID_SIGNATURE', mode: 'stripe|generic|none' }
     on 401 to aid debugging.

2. POSTGRES / SQLITE DUAL-MODE DATABASE  [COMPLETE]
   File: src/db/database.ts

   Problem (v5): createDatabase() activated the Postgres path whenever
   DATABASE_URL was set, regardless of the URL scheme.  The sandbox
   environment has DATABASE_URL=mysql://... which caused pg-pool to
   attempt a MySQL connection and throw AggregateError, breaking all
   97 tests.

   Fix (v6):
   - Added isPostgres guard: Postgres path activates only when
     DATABASE_URL starts with "postgres://" or "postgresql://".
   - All other DATABASE_URL values (mysql://, sqlite://, etc.) silently
     fall through to in-memory SQLite.
   - No changes to the Postgres schema or adapter logic.

   Production usage (unchanged): set DATABASE_URL=postgres://... to use
   Postgres; omit or set to anything else to use in-memory SQLite.
